This guide covers every button, field, and behavior in the SQL Cor
terminal. It assumes you know basic SQL.

## Opening SQL Cor

After logging in to Creatio, open the main navigation menu and
click **SQL Terminal**.

If you don't see SQL Terminal, you either haven't been granted
access or you need to refresh your browser.

## The toolbar

The toolbar contains four controls:

- **Execute** — runs the SQL in the editor
- **Clear** — empties the editor
- **Dry Run** — toggles safe-test mode
- **Database indicator** — shows the connected database

> [!important] Dry Run is your friend
> Always use Dry Run when testing DELETE or UPDATE statements.
> It runs the query inside a transaction that is always rolled
> back — so you see the row count without actually changing data.
