SQL Cor installs as a standard Creatio package. The whole process
takes about ten minutes.

## Prerequisites

You need Creatio 8.0 or newer, either Cloud or On-Premise. SQL Cor
supports PostgreSQL and MSSQL databases.

> [!note]
> SQL Cor does not require any external services. Everything runs
> inside your Creatio environment.

## Steps

### 1. Download the package

Open the GitHub releases page and download the latest
`DnSqlTerminal.zip` file.

### 2. Install via Creatio

In Creatio, navigate to **Studio → Configuration**. Drag-and-drop
the ZIP file into the package list. Wait for the installation to
complete.

> [!tip]
> A typical installation takes 5–10 minutes. If it takes longer than
> 20 minutes, check the Creatio logs for errors.

### 3. Grant initial access

Open the new **SQL Terminal** page. As a System Administrator,
you'll see the Administration button — use it to grant your team
appropriate access levels.

> [!warning]
> By default, no users have SQL Cor access. You must explicitly
> grant access through the Administration panel before anyone can
> run queries.

## Verification

Run a simple query to confirm the installation works:

```sql
SELECT version();
```

## Access levels at a glance

| Level | What it allows |
|-------|----------------|
| ReadOnly | SELECT only |
| DML | SELECT + INSERT + UPDATE + DELETE |
| DDL | Everything including CREATE / ALTER / DROP |

> [!danger]
> Never grant DDL access to a user who does not need it. DDL
> includes the ability to drop tables and modify schema.
