SQL Cor встановлюється як стандартний пакет Creatio. Увесь процес
займає близько десяти хвилин.

## Передумови

Потрібен Creatio 8.0 або новіший — Cloud або On-Premise. SQL Cor
підтримує бази PostgreSQL та MSSQL.

> [!note]
> SQL Cor не потребує зовнішніх сервісів. Усе працює всередині
> вашого середовища Creatio.

## Кроки

### 1. Завантажте пакет

Відкрийте сторінку релізів на GitHub і завантажте останній файл
`DnSqlTerminal.zip`.

### 2. Встановіть через Creatio

У Creatio перейдіть до **Studio → Configuration**. Перетягніть
ZIP-файл у список пакетів. Дочекайтеся завершення встановлення.

> [!tip]
> Типове встановлення займає 5–10 хвилин. Якщо довше 20 хвилин —
> перевірте журнали Creatio на помилки.

### 3. Надайте початковий доступ

Відкрийте нову сторінку **SQL Terminal**. Як системний адміністратор
ви побачите кнопку Administration — надайте команді відповідні
рівні доступу.

> [!warning]
> За замовчуванням жоден користувач не має доступу до SQL Cor.
> Доступ потрібно явно надати через панель Administration, перш ніж
> хтось зможе виконувати запити.

## Перевірка

Виконайте простий запит, щоб переконатися, що встановлення працює:

```sql
SELECT version();
```

## Рівні доступу

| Рівень | Можливості |
|--------|------------|
| ReadOnly | Лише SELECT |
| DML | SELECT + INSERT + UPDATE + DELETE |
| DDL | Усе, включно з CREATE / ALTER / DROP |

> [!danger]
> Не надавайте DDL користувачам, яким це не потрібно. DDL
> дозволяє видаляти таблиці та змінювати схему.
